Quick Start Examples

Generate Your First SSH Key

Create a simple Ed25519 key for general use:

$ sshman generate --algo ed25519 --use personal
‚úì SSH key generated successfully!
  Private key: /home/user/.ssh/id_personal_ed25519
  Public key:  /home/user/.ssh/id_personal_ed25519.pub

Next steps:
  1. Copy your public key: cat ~/.ssh/id_personal_ed25519.pub
  2. Add it to your remote server or service
  3. Test the connection: ssh -i ~/.ssh/id_personal_ed25519 user@server

List All Your Keys

See what keys you already have:

GitHub Setup

Create a Key for GitHub

Generate an Ed25519 key specifically for GitHub with a descriptive comment:

$ sshman generate --algo ed25519 --use github --comment "john.doe@example.com"
‚úì SSH key generated successfully!
  Private key: /home/user/.ssh/id_github_ed25519
  Public key:  /home/user/.ssh/id_github_ed25519.pub

Add the Key to GitHub

Step 1: Copy your public key

cat ~/.ssh/id_github_ed25519.pub | pbcopy  # macOS
cat ~/.ssh/id_github_ed25519.pub | xclip -selection clipboard  # Linux

Step 2: Add to GitHub

  1. Go to GitHub Settings ‚Üí SSH and GPG keys
  2. Click "New SSH key"
  3. Paste your public key
  4. Give it a descriptive title (e.g., "Work Laptop")

Step 3: Configure SSH

Add to your ~/.ssh/config:

Host github.com
    HostName github.com
    User git
    IdentityFile ~/.ssh/id_github_ed25519
    IdentitiesOnly yes

Step 4: Test the connection

$ ssh -T git@github.com
Hi username! You've successfully authenticated, but GitHub does not provide shell access.

Work Environment Setup

Secure Work Key with Passphrase

Create a highly secure RSA key for production servers:

$ sshman generate --algo rsa --bits 4096 --use work-prod --passphrase --comment "prod-access"
Enter passphrase (empty for no passphrase):
Enter same passphrase again:
‚úì SSH key generated successfully!
  Private key: /home/user/.ssh/id_work-prod_rsa
  Public key:  /home/user/.ssh/id_work-prod_rsa.pub

Organize Keys by Project

Use folder structure to keep project-specific keys organized:

Step 1: Create keys for different projects

sshman generate --use work/client-a --algo ed25519
sshman generate --use work/client-b --algo ed25519
sshman generate --use work/internal --algo ed25519

Step 2: List all keys with folder structure

$ sshman list
SSH Keys in /home/user/.ssh:

  work/client-a/id_client-a_ed25519 (+ .pub)
  work/client-b/id_client-b_ed25519 (+ .pub)
  work/internal/id_internal_ed25519 (+ .pub)

Total: 3 key(s)

Step 3: Configure SSH for each project

Add to ~/.ssh/config:

Host client-a-server
    HostName server1.client-a.com
    User deploy
    IdentityFile ~/.ssh/work/client-a/id_client-a_ed25519

Host client-b-server
    HostName server1.client-b.com
    User deploy
    IdentityFile ~/.ssh/work/client-b/id_client-b_ed25519

Deploy Key to Multiple Servers

Step 1: Generate the key

sshman generate --algo ed25519 --use work/servers

Step 2: Copy to servers

ssh-copy-id -i ~/.ssh/work/servers/id_servers_ed25519 user@server1.example.com
ssh-copy-id -i ~/.ssh/work/servers/id_servers_ed25519 user@server2.example.com
ssh-copy-id -i ~/.ssh/work/servers/id_servers_ed25519 user@server3.example.com

Step 3: Configure SSH for easy access

Add to ~/.ssh/config:

Host server1
    HostName server1.example.com
    User deploy
    IdentityFile ~/.ssh/work/servers/id_servers_ed25519

Host server2
    HostName server2.example.com
    User deploy
    IdentityFile ~/.ssh/work/servers/id_servers_ed25519

Host server3
    HostName server3.example.com
    User deploy
    IdentityFile ~/.ssh/work/servers/id_servers_ed25519

Step 4: Connect easily

ssh server1
ssh server2
ssh server3

Multiple Accounts

Personal and Work GitHub Accounts

Manage multiple GitHub accounts on the same machine:

Step 1: Generate keys for each account

$ sshman generate --algo ed25519 --use github-personal --comment "personal@gmail.com"
$ sshman generate --algo ed25519 --use github-work --comment "work@company.com"

Step 2: Configure SSH with different hosts

Edit ~/.ssh/config:

# Personal GitHub
Host github.com-personal
    HostName github.com
    User git
    IdentityFile ~/.ssh/id_github-personal_ed25519
    IdentitiesOnly yes

# Work GitHub
Host github.com-work
    HostName github.com
    User git
    IdentityFile ~/.ssh/id_github-work_ed25519
    IdentitiesOnly yes

Step 3: Clone repositories with the correct account

# Personal repositories
git clone git@github.com-personal:username/personal-repo.git

# Work repositories
git clone git@github.com-work:company/work-repo.git

Step 4: Configure git for existing repos

# For personal repo
cd personal-repo
git remote set-url origin git@github.com-personal:username/personal-repo.git

# For work repo
cd work-repo
git remote set-url origin git@github.com-work:company/work-repo.git

Server Deployment

CI/CD Pipeline Key

Create a key for automated deployments (no passphrase for automation):

$ sshman generate --algo ed25519 --use ci-deploy --comment "ci-bot@company.com"
‚úì SSH key generated successfully!
  Private key: /home/user/.ssh/id_ci-deploy_ed25519
  Public key:  /home/user/.ssh/id_ci-deploy_ed25519.pub
Security Note: For CI/CD keys, use strict server-side restrictions:
  • Limit to specific commands with command= in authorized_keys
  • Restrict source IPs with from= option
  • Use dedicated service accounts with minimal permissions

Bastion Host Setup

Step 1: Create bastion key

sshman generate --algo ed25519 --use bastion --passphrase

Step 2: Configure SSH with ProxyJump

Edit ~/.ssh/config:

Host bastion
    HostName bastion.company.com
    User admin
    IdentityFile ~/.ssh/id_bastion_ed25519

Host internal-server
    HostName 10.0.1.100
    User deploy
    ProxyJump bastion
    IdentityFile ~/.ssh/id_work-servers_ed25519

Step 3: Connect through bastion

$ ssh internal-server
# Automatically connects through bastion

Key Rotation

Rotate an Existing Key

Best practice: Rotate keys every 6-12 months

Step 1: Check current key info

$ sshman info id_work_ed25519
SSH Key Information: id_work_ed25519

Key Details:
  Type: Ed25519
  Size: 256 bits
  Comment: user@hostname
  Created: 2024-01-15

Fingerprints:
  SHA256: SHA256:abc123def456...

Step 2: Generate new key with updated tag

$ sshman generate --algo ed25519 --use work --comment "rotated-2025-12"
‚ö† Warning: Key 'id_work_ed25519' already exists
Use --force to overwrite or choose a different name

$ sshman generate --algo ed25519 --use work --comment "rotated-2025-12" --force
‚úì SSH key generated successfully!
  Private key: /home/user/.ssh/id_work_ed25519
  Public key:  /home/user/.ssh/id_work_ed25519.pub

Step 3: Update on all services

# Copy new public key
cat ~/.ssh/id_work_ed25519.pub

# Update on:
# - GitHub/GitLab
# - All servers
# - Cloud providers
# - Any other services

Step 4: Verify and test

# Test each service
ssh -T git@github.com
ssh user@server.example.com

# If all works, the old key has been replaced

Team Workflow

Onboarding New Team Member

Step 1: Team member generates their key

sshman generate --algo ed25519 --use company --comment "jane.smith@company.com" --passphrase

Step 2: Share public key with admin

# Email or Slack
cat ~/.ssh/id_company_ed25519.pub

Step 3: Admin adds key to all servers

# On each server
echo "ssh-ed25519 AAAAC3... jane.smith@company.com" >> ~/.ssh/authorized_keys

Step 4: Team member tests access

ssh user@dev-server.company.com
ssh user@staging-server.company.com

Key Audit Across Team

Regular security audit to review all SSH keys:

$ sshman list
Found 5 SSH keys in /home/user/.ssh:

# Review each key
$ sshman info id_github_ed25519
$ sshman info id_work_rsa
$ sshman info id_personal_ed25519

# Create audit report
$ sshman list > ssh-keys-audit-$(date +%Y%m%d).txt

üí° Pro Tips

Use SSH Agent

Add keys to ssh-agent to avoid typing passphrases repeatedly:

eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_work_ed25519

Backup Your Keys

Keep encrypted backups of your private keys in a secure location:

tar -czf ssh-backup.tar.gz ~/.ssh/
gpg -c ssh-backup.tar.gz

Document Your Keys

Keep a record of which keys are used where:

sshman list > ~/.ssh/KEY_INVENTORY.txt

Use SSH Config

Create shortcuts in ~/.ssh/config for easier access:

Host prod
    HostName prod.example.com
    User deploy
    IdentityFile ~/.ssh/id_work_ed25519

Usage Guide

Overview

sshman is a command-line tool for managing SSH keys and connections. It provides seven main commands:

  • generate - Create new SSH key pairs
  • use - Start ssh-agent and add SSH keys (with auto-clear)
  • init - Create .sshman file for auto-directory switching
  • connect-new - Create new SSH connection profiles interactively
  • connect - Connect to saved SSH profiles by alias
  • list - Display all SSH keys
  • info - Show detailed information about a specific key
Note: All commands support --help flag for detailed usage information.

Generate Command

Create new SSH key pairs with customizable options.

Basic Syntax

sshman generate [OPTIONS]

Options

Option Description Default
-a, --algo <ALGORITHM> Key algorithm (rsa, dsa, ecdsa, ed25519) ed25519
-b, --bits <SIZE> Key size in bits Algorithm-dependent
-u, --use <TAG> Use case/folder structure (e.g., 'work', 'personal', 'work/project-a') None
-n, --name <NAME> Custom key name id_<use>_<algo>
-c, --comment <TEXT> Comment for the key user@hostname
-p, --passphrase Prompt for passphrase protection No passphrase
-o, --output <DIR> Output directory ~/.ssh
-f, --force Overwrite existing keys false

Examples

Generate Ed25519 key for work

sshman generate --algo ed25519 --use work

Creates: ~/.ssh/work/id_work_ed25519 and ~/.ssh/work/id_work_ed25519.pub

Generate key in nested folder structure

sshman generate --algo ed25519 --use work/project-a

Creates: ~/.ssh/work/project-a/id_project-a_ed25519 and corresponding .pub file

Generate RSA key with custom size

sshman generate --algo rsa --bits 4096 --use github

Creates a 4096-bit RSA key for GitHub

Generate with passphrase and custom comment

sshman generate --algo ed25519 --use personal --passphrase --comment "john@mycompany.com"

Prompts for passphrase and adds custom comment

Generate with fully custom name

sshman generate --name my_special_key --algo ecdsa --bits 521

Creates: my_special_key and my_special_key.pub

Use Command

Start ssh-agent and add SSH keys with a single command. Automatically clears existing keys to prevent authentication conflicts.

Basic Syntax

eval "$(sshman use <KEY_NAME> --quiet)"

Options

Option Description Default
-q, --quiet Output only the command for use with eval false
-t, --time <SECONDS> Lifetime for the key in ssh-agent (in seconds) Unlimited
--path <DIR> Custom SSH directory path ~/.ssh

How It Works

  • Detects if ssh-agent is already running
  • Clears all existing keys before adding the new one (prevents conflicts)
  • If no agent is running, starts a new ssh-agent
  • Adds the specified key to the agent

Examples

Basic usage

eval "$(sshman use work/github/id_github_ed25519 --quiet)"

Loads the work GitHub key, clearing any existing keys first

With time limit (1 hour)

eval "$(sshman use personal/id_personal_ed25519 --quiet --time 3600)"

Key expires from agent after 1 hour

View the command without executing

sshman use work/key

Shows what would be executed without --quiet flag

Note: When using an existing ssh-agent, all currently loaded keys are automatically cleared before adding the new key. This ensures only the intended key is active and prevents authentication conflicts when working across different projects.

Init Command

Create a .sshman file in the current directory for automatic SSH key loading when you enter the directory.

Basic Syntax

sshman init <KEY_NAME> [OPTIONS]

Options

Option Description
-f, --force Overwrite existing .sshman file
--path <DIR> Custom SSH directory path (default: ~/.ssh)

How It Works

The init command creates a .sshman file in your project directory. When you have shell hooks installed, the key is automatically loaded when you cd into that directory.

Setup Shell Hooks

cd completions
./install.sh

# Reload your shell
source ~/.bashrc  # or ~/.zshrc

Examples

Initialize for a project

cd ~/projects/work-project
sshman init work/gitlab/id_gitlab_ed25519

Creates .sshman file. Key auto-loads when entering this directory

Update existing .sshman file

sshman init work/new-key --force

Overwrites existing .sshman file with new key

Important: .sshman files should be added to your .gitignore as they contain user-specific SSH key preferences.

Connect-New Command

Create a new SSH connection profile interactively. Profiles are saved for quick access via the connect command.

Basic Syntax

sshman connect-new

Interactive Prompts

The command will prompt you for the following information:

  • Alias (required) - A unique name for this profile (e.g., "production-server", "dev-db")
  • Hostname (required) - Server hostname or IP address
  • Username (required) - SSH username for the connection
  • Port (optional) - SSH port number (default: 22)
  • SSH key path (optional) - Path to SSH private key (shows available keys from ~/.ssh)

Features

  • Validates that SSH key exists before saving
  • Shows available SSH keys from ~/.ssh directory
  • Supports both absolute and relative paths for SSH keys
  • Prevents duplicate aliases

Example Session

$ sshman connect-new
Create a new SSH connection profile

Alias: production-server
Hostname: prod.example.com
Username: deploy
Port (default: 22): 22
Available SSH keys in /home/user/.ssh:
  - id_ed25519
  - work/id_work_ed25519
  - personal/id_personal_ed25519

SSH key path (optional): work/id_work_ed25519

Profile created successfully!
  Alias:    production-server
  Hostname: prod.example.com
  Username: deploy
  Port:     22
  SSH Key:  /home/user/.ssh/work/id_work_ed25519

To connect, run:
  sshman connect production-server
Storage: Profiles are stored in ~/.sshman/profiles.json

Connect Command

Connect to a saved SSH profile by its alias. Supports tab completion for profile names.

Basic Syntax

sshman connect <ALIAS>

How It Works

  • Looks up the profile by alias in ~/.sshman/profiles.json
  • Constructs the SSH command with saved parameters
  • Executes the SSH connection interactively
  • Shows the actual SSH command before connecting

Examples

Connect to a saved profile

$ sshman connect production-server
Connecting to production-server...
Command: ssh -p 22 -i /home/user/.ssh/work/id_work_ed25519 deploy@prod.example.com

# You're now connected to the remote server

Profile not found

$ sshman connect nonexistent
Profile not found: nonexistent

Available profiles:
  - production-server (deploy@prod.example.com)
  - dev-db (root@192.168.1.100)

To create a new profile, run:
  sshman connect-new
Tab Completion: Profile aliases support tab completion when shell completion is installed.

List Command

Display all SSH keys found in your ~/.ssh directory.

Basic Syntax

sshman list [OPTIONS]

Options

Option Description
-a, --algo <ALGORITHM> Filter by algorithm (rsa, dsa, ecdsa, ed25519)
-d, --dir <PATH> Custom SSH directory (default: ~/.ssh)

Output Format

For each key found, the list command displays:

  • Key name
  • Algorithm type
  • Key size (bits)
  • SHA256 fingerprint
  • Public key location

Examples

List all SSH keys

sshman list

List only Ed25519 keys

sshman list --algo ed25519

List keys from custom directory

sshman list --dir /custom/path/to/keys

Info Command

Display comprehensive information about a specific SSH key.

Basic Syntax

sshman info <KEY_NAME> [OPTIONS]

Options

Option Description
-d, --dir <PATH> Custom SSH directory (default: ~/.ssh)

Output Information

  • Key type and algorithm
  • Key size in bits
  • Comment/identifier
  • MD5 fingerprint
  • SHA256 fingerprint
  • File locations (private and public keys)
  • File permissions

Examples

View info for work key

sshman info id_work_ed25519

View info with key in custom directory

sshman info my_key --dir /custom/keys
Note: Provide the key name without the .pub extension.

SSH Key Algorithms

sshman supports four SSH key algorithms, each with different characteristics:

Ed25519 (Recommended)

  • Size: 256 bits (fixed)
  • Security: Excellent
  • Speed: Very fast
  • Compatibility: Modern systems (OpenSSH 6.5+)

Use when: You need the best security and performance on modern systems.

RSA

  • Size: 2048-4096 bits (recommended: 4096)
  • Security: Good (with 4096 bits)
  • Speed: Moderate
  • Compatibility: Universal

Use when: You need maximum compatibility with older systems.

ECDSA

  • Size: 256, 384, or 521 bits
  • Security: Good
  • Speed: Fast
  • Compatibility: Most modern systems

Use when: You want a balance between Ed25519 and RSA.

DSA (Deprecated)

  • Size: 1024 bits (fixed)
  • Security: Weak (deprecated)
  • Compatibility: Legacy systems only

Use when: Only for legacy systems that don't support other algorithms.

Troubleshooting

‚ùå "Key already exists"

Solution: Use the --force flag to overwrite, or choose a different name.

sshman generate --use work --force

‚ùå "ssh-keygen not found"

Solution: Install OpenSSH tools:

# Ubuntu/Debian
sudo apt-get install openssh-client

# macOS (usually pre-installed)
brew install openssh

‚ùå "Permission denied"

Solution: Check file permissions:

ls -la ~/.ssh/
chmod 600 ~/.ssh/id_*

‚ùå "Key not found"

Solution: Make sure you're using the correct key name without the .pub extension:

# ‚ùå Wrong
sshman info id_work_ed25519.pub

# ‚úÖ Correct
sshman info id_work_ed25519

‚ùå "Invalid key size"

Solution: Use valid key sizes for each algorithm:

  • RSA: 2048, 3072, 4096
  • DSA: 1024 (fixed)
  • ECDSA: 256, 384, 521
  • Ed25519: 256 (fixed)

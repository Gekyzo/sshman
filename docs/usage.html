Usage Guide

Overview

sshman is a command-line tool for managing SSH keys. It provides three main commands:

  • generate - Create new SSH key pairs
  • list - Display all SSH keys
  • info - Show detailed information about a specific key
Note: All commands support --help flag for detailed usage information.

Generate Command

Create new SSH key pairs with customizable options.

Basic Syntax

sshman generate [OPTIONS]

Options

Option Description Default
-a, --algo <ALGORITHM> Key algorithm (rsa, dsa, ecdsa, ed25519) ed25519
-b, --bits <SIZE> Key size in bits Algorithm-dependent
-u, --use <TAG> Use case tag (work, personal, github, etc.) None
-n, --name <NAME> Custom key name id_<use>_<algo>
-c, --comment <TEXT> Comment for the key user@hostname
-p, --passphrase Prompt for passphrase protection No passphrase
-o, --output <DIR> Output directory ~/.ssh
-f, --force Overwrite existing keys false

Examples

Generate Ed25519 key for work

sshman generate --algo ed25519 --use work

Creates: id_work_ed25519 and id_work_ed25519.pub

Generate RSA key with custom size

sshman generate --algo rsa --bits 4096 --use github

Creates a 4096-bit RSA key for GitHub

Generate with passphrase and custom comment

sshman generate --algo ed25519 --use personal --passphrase --comment "john@mycompany.com"

Prompts for passphrase and adds custom comment

Generate with fully custom name

sshman generate --name my_special_key --algo ecdsa --bits 521

Creates: my_special_key and my_special_key.pub

List Command

Display all SSH keys found in your ~/.ssh directory.

Basic Syntax

sshman list [OPTIONS]

Options

Option Description
-a, --algo <ALGORITHM> Filter by algorithm (rsa, dsa, ecdsa, ed25519)
-d, --dir <PATH> Custom SSH directory (default: ~/.ssh)

Output Format

For each key found, the list command displays:

  • Key name
  • Algorithm type
  • Key size (bits)
  • SHA256 fingerprint
  • Public key location

Examples

List all SSH keys

sshman list

List only Ed25519 keys

sshman list --algo ed25519

List keys from custom directory

sshman list --dir /custom/path/to/keys

Info Command

Display comprehensive information about a specific SSH key.

Basic Syntax

sshman info <KEY_NAME> [OPTIONS]

Options

Option Description
-d, --dir <PATH> Custom SSH directory (default: ~/.ssh)

Output Information

  • Key type and algorithm
  • Key size in bits
  • Comment/identifier
  • MD5 fingerprint
  • SHA256 fingerprint
  • File locations (private and public keys)
  • File permissions

Examples

View info for work key

sshman info id_work_ed25519

View info with key in custom directory

sshman info my_key --dir /custom/keys
Note: Provide the key name without the .pub extension.

SSH Key Algorithms

sshman supports four SSH key algorithms, each with different characteristics:

Ed25519 (Recommended)

  • Size: 256 bits (fixed)
  • Security: Excellent
  • Speed: Very fast
  • Compatibility: Modern systems (OpenSSH 6.5+)

Use when: You need the best security and performance on modern systems.

RSA

  • Size: 2048-4096 bits (recommended: 4096)
  • Security: Good (with 4096 bits)
  • Speed: Moderate
  • Compatibility: Universal

Use when: You need maximum compatibility with older systems.

ECDSA

  • Size: 256, 384, or 521 bits
  • Security: Good
  • Speed: Fast
  • Compatibility: Most modern systems

Use when: You want a balance between Ed25519 and RSA.

DSA (Deprecated)

  • Size: 1024 bits (fixed)
  • Security: Weak (deprecated)
  • Compatibility: Legacy systems only

Use when: Only for legacy systems that don't support other algorithms.

Troubleshooting

‚ùå "Key already exists"

Solution: Use the --force flag to overwrite, or choose a different name.

sshman generate --use work --force

‚ùå "ssh-keygen not found"

Solution: Install OpenSSH tools:

# Ubuntu/Debian
sudo apt-get install openssh-client

# macOS (usually pre-installed)
brew install openssh

‚ùå "Permission denied"

Solution: Check file permissions:

ls -la ~/.ssh/
chmod 600 ~/.ssh/id_*

‚ùå "Key not found"

Solution: Make sure you're using the correct key name without the .pub extension:

# ‚ùå Wrong
sshman info id_work_ed25519.pub

# ‚úÖ Correct
sshman info id_work_ed25519

‚ùå "Invalid key size"

Solution: Use valid key sizes for each algorithm:

  • RSA: 2048, 3072, 4096
  • DSA: 1024 (fixed)
  • ECDSA: 256, 384, 521
  • Ed25519: 256 (fixed)
